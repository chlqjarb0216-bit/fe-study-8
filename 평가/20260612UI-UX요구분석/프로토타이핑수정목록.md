# 목록페이지

1. 게시글의 수정을 허용했을 경우 작성일과 번호로 정렬을 시키면 수정한 내용이 게시판 위로 올라오지 않아 수정한 내용이 그대로 묻힐 가능성이 존재. 작성일이 아닌 최종수정일을 보여주고 이를 기반으로 정렬하는 편이 좋아보임.
2. 조회수 외에 게시글에 추천을 줄 수 있는 기능을 구현해 표시하는 것으로 인기있는(유용한) 게시글에 접속하기 쉽게 만들면 좋을 것 같다. 혹은 댓글 기능을 구현해 댓글 수를 표시하는 것도 좋아보인다.
3. 검색기능의 구현. 작성자 혹은 제목을 기반으로한 검색기능으로 사용자가 원하는 정보에 더 쉽게 접근할 수 있도록 함.
4. 필터기능. 2번의 추천수 및 댓글수 를 기반으로한 필터를 적용하는 것으로 사용자 편의성을 증진시킬 수 있음.


# 상세페이지

1. 목록페이지에서 언급한 추천 및 댓글작성 기능의 구현.
2. 게시글 수정 버튼을 만들어 글작성자가 맞다면 게시글 수정페이지로 넘어가는 기능이 필요.
3. 마찬가지로 글작성자의 권한으로 게시글의 삭제가 가능할 수 있도록 함.
4. 본문 내용 댓글 영역 아래쪽에 위의 목록페이지의 연장을 보여주는 것으로 사용자가 게시글을 열람하고 다시 목록페이지로 나가야하는 번거로움을 줄일 수 있음.
